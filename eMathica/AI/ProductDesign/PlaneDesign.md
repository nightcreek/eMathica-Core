## 1. 产品定位

eMathica 的 Plane Calculator 不应被定义为普通二维函数计算器，也不应只是几何画板。

更准确的定位是：

> Plane Calculator 是一个默认界面克制的二维数学艺术画布。  
> 它保留传统几何画板 / 函数画板的入口习惯，但底层支持函数、几何、列表、手绘、拟合、区域填色、参数、依赖、样式渐变和基础动画。

Plane Calculator 的目标不是只让用户“算出结果”或“画出标准图像”，而是让用户能够：

- 创建二维数学对象
- 组织对象之间的数学关系
- 使用函数、列表、参数和手绘路径生成图案
- 调整颜色、透明度、笔触、渐变和填充
- 让作品随参数或时间变化
- 导出可用于展示、分享或继续创作的数学艺术作品

---

## 2. Plane 与共享系统的职责边界

Plane 不应承担所有通用能力。  
Plane 只是 eMathica 通用数学创作系统在二维平面中的实现层。

### 2.1 Plane 特有职责

Plane 主要负责：

- 二维坐标画布
- 二维坐标系显示
- 二维对象渲染
- 二维对象命中测试
- 二维对象拖拽与交互
- 二维几何构造算法
- 二维函数 / 曲线采样后的呈现
- 二维区域填色
- 二维对象在动画播放时的显示
- 二维导出渲染

### 2.2 不应属于 Plane 私有的能力

以下能力应尽量作为共享能力处理：

- 数学对象模型
- 表达式模型
- LaTeX 输入
- 结构化数学键盘
- AST / 表达式分析
- 参数系统
- 动画系统
- 样式系统
- 对象选择与编辑框架
- Inspector 框架
- 保存 / 打开 / 项目包
- 导出系统
- 模板系统
- 示例作品系统
- 笔记系统

Plane 只负责把这些共享能力在二维画布中表达出来。

### 2.3 与 Space Calculator 的关系

Plane 和 Space 应共享对象、表达式、参数、样式、动画、保存、导出等上层能力。

但 Plane 和 Space 不应强行共享具体几何算法：

- Plane 负责 2D 几何和 2D 渲染
- Space 负责 3D 几何、投影、相机和 3D 渲染
- 二者可以共享概念，但不必共享所有实现

---

## 3. 默认界面原则

Plane Calculator 默认打开时，应接近用户熟悉的几何画板 / 函数画板，不应一次性展示过多内容。

### 3.1 默认界面结构

默认状态建议：

| 区域 | 状态 |
|---|---|
| 中央 | 标准二维坐标画布 |
| 工具栏 | 选择、移动、点、线、圆、函数等基础工具 |
| 输入入口 | 函数 / 表达式输入 |
| 对象与表达式面板 | 默认收起或轻量显示 |
| Inspector | 选中对象后出现 |
| 样式面板 | 需要时展开 |
| 动画控制 | 有参数播放时出现 |

### 3.2 默认体验目标

用户打开 Plane 后应首先感受到：

- 这是一个干净的二维数学画布
- 可以马上输入函数
- 可以马上创建点、线、圆等对象
- 可以拖动、缩放、选择对象
- 高级能力存在，但不打扰初始使用

默认界面不应给用户“复杂专业设计软件”的压迫感。

---

## 4. 坐标系设计

Plane Calculator 采用清晰的标准数学坐标系，不采用手绘坐标系或艺术坐标系。

### 4.1 应支持

- 标准坐标轴
- 标准网格
- 极简坐标轴
- 隐藏坐标轴
- 隐藏网格
- 透明背景
- 导出时单独控制坐标轴 / 网格是否显示

### 4.2 不放入 Plane 的方向

- 手绘坐标系
- 艺术坐标系
- 过度装饰化坐标轴

数学艺术效果应主要由对象、颜色、透明度、渐变、动画、区域填色和构图来实现，而不是通过复杂坐标系实现。

---

## 5. 核心对象类型

Plane 的对象不应只按“几何对象”理解。  
完整设计中，Plane 应支持多类二维创作对象。

### 5.1 几何对象

包括：

- 点
- 线段
- 直线
- 射线
- 圆
- 圆弧
- 多边形
- 角
- 向量

几何对象应支持：

- 创建
- 选择
- 拖拽
- 命名
- 样式
- 依赖关系
- 保存 / 恢复
- 导出

### 5.2 函数与曲线对象

包括：

- 显函数
- 反向显函数
- 参数曲线
- 极坐标曲线
- 隐式曲线
- 分段函数
- 不等式区域
- 点列图
- 折线图
- 曲线族

函数与曲线是 Plane 的重点能力之一。  
Plane 应加强函数相关能力，使其不只是普通几何画板。

### 5.3 列表与表格对象

包括：

- 数值列表
- 点列表
- 参数列表
- 对象列表
- 颜色列表
- 表格
- 点阵
- 曲线族
- 圆阵列
- 列表生成区域

列表不只是数据结构，而是数学艺术创作中的批量生成和批量控制工具。

### 5.4 手绘对象

包括：

- Apple Pencil 背景笔迹
- 草图
- 装饰线条
- 手写标注
- 简单背景绘画

手绘对象默认不参与数学计算，也不自动转换为函数。

### 5.5 拟合对象

包括：

- 平滑路径
- 点列采样
- 拟合函数
- 拟合参数曲线
- 样条曲线
- 多项式拟合曲线
- 傅里叶 / 级数拟合曲线
- 几何识别对象

拟合对象由用户主动从手绘笔迹、点列或数据中转换生成。

### 5.6 区域对象

包括：

- 多曲线封闭区域
- 两曲线之间区域
- 不等式区域
- 手绘路径区域
- 列表生成区域
- 智能封口区域
- 遮罩区域

区域对象用于填色、透明度渐变、构图和导出。

### 5.7 艺术表现对象

包括：

- 轨迹
- 残影
- 渐变曲线
- 透明度渐变区域
- 背景层
- 遮罩
- 高亮对象
- 动画路径

这些对象不一定都有独立工具，但应在能力地图中保留。

## 6. 函数与曲线能力

Plane 的函数系统应吸收类似 Desmos 的表达式驱动能力，但不应完全照搬 Desmos 的交互形态。

### 6.1 需要支持的函数类型

完整设计中应支持：

| 类型 | 说明 |
|---|---|
| 显函数 | 普通 y = f(x) |
| 反向显函数 | x = f(y) |
| 参数曲线 | x(t), y(t) |
| 极坐标曲线 | r(θ) |
| 隐式曲线 | F(x, y) = 0 |
| 分段函数 | 不同区间不同表达式 |
| 不等式区域 | 条件定义的填色区域 |
| 点列 / 折线 | 离散点与连接线 |
| 曲线族 | 由列表或参数批量生成 |

### 6.2 函数管理能力

函数应支持：

- 命名
- 引用其他函数
- 引用参数
- 引用列表
- 隐藏 / 显示
- 分组
- 注释
- 样式修改
- 笔触选择
- 参与动画
- 参与区域填色
- 参与导出

### 6.3 限制与裁剪

应支持：

- x 范围限制
- y 范围限制
- 参数范围限制
- 区域限制
- 条件显示
- 分段显示
- 两曲线之间填色
- 列表控制多个显示区间

限制与裁剪是数学艺术创作中的基础能力，不只是高级数学功能。

### 6.4 函数笔触

函数对象的数学定义应保持精确，但渲染风格可以艺术化。

函数可选择的笔触方向包括：

- 标准线
- 手绘线
- 铅笔线
- 荧光线
- 点线
- 渐变线
- 轨迹线

函数笔触不改变数学定义，只改变视觉表现。

---

## 7. 列表系统

列表是 Plane 未来的重要核心能力之一。

### 7.1 列表的定位

列表不只是数据容器。  
它应作为批量生成、批量控制和数学艺术构图的工具。

### 7.2 列表用途

列表应支持或预留：

| 用途 | 说明 |
|---|---|
| 控制函数取值范围 | 用列表定义多个显示区间 |
| 控制曲线参数范围 | 用列表定义多段参数范围 |
| 批量生成点 | 点阵、采样点 |
| 批量生成曲线 | 曲线族 |
| 批量生成圆 | 圆阵列 |
| 批量生成线段 | 连接结构 |
| 批量生成区域 | 区域族 |
| 控制颜色 | 列表配色 |
| 控制透明度 | 批量透明度渐变 |
| 控制线宽 | 批量线宽变化 |
| 控制动画参数 | 多对象动态变化 |

### 7.3 列表与艺术效果

列表可用于：

- 曲线族渐变
- 圆阵列
- 点阵
- 波纹
- 花纹
- 轨迹帧
- 多段函数显示
- 多区域填色
- 按编号控制透明度
- 按编号控制颜色

### 7.4 第一版与后续关系

第一版可以先支持基础列表对象和少量批量生成能力。  
后续应加强：

- 列表控制函数取值范围
- 列表控制曲线参数范围
- 列表生成区域
- 列表驱动样式
- 列表驱动动画

---

## 8. 对象与表达式面板

Plane 不应分别做复杂的表达式列表、对象列表和图层列表。  
应优先设计统一的“对象与表达式面板”。

### 8.1 面板内容

面板中可以包含：

- 函数
- 参数
- 几何对象
- 列表与表格
- 手绘与拟合
- 区域与填充
- 分组
- 注释

### 8.2 分类方式

分类分为两层：

#### 自动分类

系统按对象类型自动归类，例如：

- 函数
- 参数
- 几何
- 列表
- 表格
- 手绘
- 拟合
- 区域
- 样式 / 辅助

#### 用户分类

用户可以自己创建：

- 文件夹
- 对象组
- 图层组
- 构造步骤组
- 背景组
- 导出组
- 隐藏组

### 8.3 面板原则

- 默认不展开太多内容
- 复杂对象可折叠
- 选中对象后 Inspector 展示详细属性
- 面板应服务整理，不应抢占画布主体
- 用户可以按自己的创作逻辑重组对象

---

## 9. Apple Pencil 设计

Apple Pencil 在 Plane 中应分为两种用途。

### 9.1 背景绘画模式

该模式用于：

- 背景装饰
- 草图辅助
- 手写标注
- 箭头
- 圈画
- 简单纹理
- 创作构图

默认不自动识别函数，也不参与数学计算。

背景绘画模式应支持：

- 画笔
- 橡皮
- 撤销
- 颜色
- 透明度
- 笔触粗细
- 背景层锁定
- 背景层显示 / 隐藏
- 背景层是否导出

### 9.2 数学识别 / 拟合模式

用户主动选择笔迹后，可以转换为数学对象。

可选转换包括：

- 平滑路径
- 点列采样
- 参数路径
- 显函数
- 参数曲线
- 多项式拟合
- 样条拟合
- 傅里叶 / 级数拟合
- 几何对象识别

### 9.3 设计原则

- Pencil 默认是创作工具，不是识别工具
- 不应自动把用户画的背景线条转换成函数
- 手绘笔迹应可保留
- 拟合结果应由用户主动选择生成
- 可以同时保留原笔迹和拟合对象
- 用户应能设置是否显示原笔迹、拟合曲线、采样点

---

## 10. 手绘拟合能力

手绘拟合是 eMathica 区别于普通函数绘图器的重要特色。

### 10.1 基本流程

建议流程：

1. 用户用 Apple Pencil 画曲线
2. 曲线先作为手绘笔迹保留
3. 用户选中笔迹
4. 系统提供转换选项
5. 用户选择拟合方式
6. 系统生成对应数学对象
7. 用户决定是否保留原笔迹

### 10.2 拟合类型

应规划以下拟合方式：

| 类型 | 用途 |
|---|---|
| 平滑路径 | 保持视觉连续 |
| 点列采样 | 作为点列表或路径基础 |
| 多项式拟合 | 适合单值曲线 |
| 样条拟合 | 适合自由艺术路径 |
| 参数曲线拟合 | 适合回环、闭合、复杂路径 |
| 傅里叶 / 级数拟合 | 适合周期、闭合、艺术曲线 |
| 分段函数拟合 | 适合局部变化明显的曲线 |
| 几何基元识别 | 识别直线、圆、圆弧、多边形 |

### 10.3 输出对象

拟合结果可生成：

- 平滑路径对象
- 点列表对象
- 显函数对象
- 参数曲线对象
- 级数曲线对象
- 几何对象
- 运动路径
- 区域边界

### 10.4 重要原则

不要强制用函数表达所有手绘曲线。  
很多手绘曲线更适合被表示为参数路径、样条路径或点列表。

---

## 11. 区域与填色系统

区域与填色是数学艺术创作的关键能力。

### 11.1 区域对象类型

应支持或规划：

- 多曲线封闭区域
- 两曲线之间区域
- 不等式区域
- 多边形区域
- 手绘路径区域
- 列表生成区域
- 智能封口区域
- 遮罩区域

### 11.2 多曲线组合区域

用户可以选择多条曲线、线段、圆弧、函数片段或拟合路径，使它们共同构成一个整体图形，用于填色。

该功能不是普通分组。

区别如下：

| 功能 | 作用 |
|---|---|
| 普通分组 | 管理多个对象 |
| 多曲线组合区域 | 多条边界形成可填色区域 |

### 11.3 区域属性

区域对象应支持：

- 边界对象
- 填充颜色
- 填充透明度
- 颜色渐变
- 透明度渐变
- 边界显示 / 隐藏
- 边界样式
- 层级
- 是否导出
- 是否参与动画
- 是否参与遮罩

### 11.4 两曲线之间填色

应支持选择两条曲线，在指定区间内生成两曲线之间的填色区域。

这既有数学意义，也有艺术构图价值。

### 11.5 不等式区域

不等式区域应作为函数系统和区域系统的交叉能力。

例如：

- 某曲线上方区域
- 某曲线下方区域
- 多个不等式交集区域
- 随参数变化的区域

---

## 12. 智能封口

用户画图时，边界不一定完全连接。  
Plane 的填色系统不应只接受严格闭合图形。

### 12.1 功能定义

智能封口指：

> 当多条边界曲线的端点相邻但没有真正连接时，系统自动建立不可见的虚拟连接边，用于填色边界判断，但不在画布上显示，也不改变原曲线。

### 12.2 边界分层

区域对象应区分：

| 边界类型 | 作用 |
|---|---|
| 可见边界 | 用户看到的曲线、线段、圆弧 |
| 虚拟封口边 | 不显示，只参与填色边界计算 |

### 12.3 用户体验

用户创建区域时，系统可以：

- 自动检测小缝隙
- 自动建立虚拟封口
- 生成填色区域
- 显示使用了几处智能封口
- 允许关闭智能封口
- 允许设置封口距离为小 / 中 / 大

### 12.4 注意事项

智能封口需要避免误判：

- 端点距离过大时不应自动连接
- 端点方向不合理时应谨慎
- 多个候选端点接近时不应随意连接
- 用户故意留白时不应强制封口
- 复杂自交区域可后置处理

### 12.5 设计原则

填色系统不要求视觉边界完全闭合。  
Plane 可以使用不可见的虚拟封口边来完成区域闭合，使短线式边界也能形成可靠填色区域，同时不破坏原始视觉效果。

---

## 13. 样式系统

样式系统是数学艺术表达层，不是附属装饰。

### 13.1 基础样式

应支持：

- 颜色
- 透明度
- 线宽
- 线型
- 点样式
- 填充色
- 填充透明度
- 边界样式
- 标签样式

### 13.2 渐变样式

应支持或规划：

- 颜色渐变
- 透明度渐变
- 线宽渐变
- 填充渐变
- 边缘羽化

### 13.3 渐变驱动方式

渐变可以由以下因素驱动：

- 沿曲线方向
- 参数值
- 距离
- 列表编号
- 时间
- 函数值
- 点运动速度
- 用户指定规则

### 13.4 透明度渐变

透明度渐变应进入正式功能地图。

典型用途：

- 曲线尾部淡出
- 运动轨迹残影
- 曲线族层次
- 圆阵列光晕
- 区域边缘羽化
- 不等式区域渐隐
- 点阵密度表现
- 动态图形呼吸效果

### 13.5 笔触样式

数学对象应能选择笔触：

- 标准线
- 手绘线
- 铅笔线
- 荧光线
- 点线
- 渐变线
- 轨迹线

数学定义保持精确，笔触只影响渲染。

## 14. 参数、依赖与变化

Plane 中的对象不应是静态图形，而应是关系节点。

### 14.1 依赖类型

应区分：

| 类型 | 说明 |
|---|---|
| 构造依赖 | 子对象由父对象构造出来 |
| 参数依赖 | 对象依赖参数变化 |
| 列表依赖 | 对象由列表批量生成或控制 |
| 样式依赖 | 样式由参数、列表、时间或对象状态控制 |
| 相对关系 | 对象相对另一个对象变化 |
| 跟随关系 | 对象跟随路径、点或曲线 |
| 动画依赖 | 对象随时间或播放状态变化 |

### 14.2 构造依赖

包括：

- 中点
- 交点
- 平行线
- 垂线
- 圆心过点圆
- 三点圆弧
- 两曲线之间区域
- 拟合曲线依赖原始笔迹
- 区域依赖边界曲线

### 14.3 相对变化

对象可以：

- 在自身基础上移动
- 在自身基础上旋转
- 在自身基础上缩放
- 相对另一个对象保持位置
- 相对另一个对象保持距离
- 相对另一个对象保持角度
- 跟随曲线或路径
- 使用其他对象的参数控制样式

### 14.4 基础状态与变化层

应概念上区分：

- 对象基础定义
- 几何变换
- 样式变化
- 动画变化
- 依赖关系

对象可以保留基础状态，再叠加变化。

### 14.5 第一阶段原则

不应一开始做完整复杂约束求解器。  
但应保证：

- 父对象变化，子对象稳定更新
- 参数变化，相关函数、几何对象和样式同步变化
- 区域边界变化，填色区域同步变化
- 手绘拟合对象可选择是否跟随原笔迹更新

## 18. 目前已确认的设计原则

### 原则 1：Plane 是二维数学艺术画布

Plane 不只是函数计算器，也不只是几何画板。

### 原则 2：默认界面克制

默认打开时接近普通几何画板，不一次性展示全部高级能力。

### 原则 3：Plane 是二维实现层

表达式、输入、参数、样式、动画、保存、导出等应尽量共享，Plane 负责二维呈现。

### 原则 4：函数能力需要加强

Plane 应吸收表达式列表、参数、限制条件、曲线族、列表等函数创作能力。

### 原则 5：列表是核心创作能力

列表不只是数据，而是批量生成对象和控制样式、范围、动画的工具。

### 原则 6：Pencil 默认是创作工具

Apple Pencil 默认用于绘画背景、草图、标注，不自动转换函数。

### 原则 7：手绘数学化必须由用户主动触发

笔迹可以保留，也可以手动转换为路径、点列、函数、参数曲线或拟合对象。

### 原则 8：数学对象可以艺术化渲染

函数和几何对象保持精确定义，但可以选择笔触、渐变、透明度和样式。

### 原则 9：曲线可以作为区域边界

多条曲线可以组合成整体图形，生成可填色区域对象。

### 原则 10：填色不要求视觉边界完全闭合

可以使用不可见的虚拟封口边完成区域闭合，防止填色外溢。

### 原则 11：透明度渐变是正式能力

透明度渐变应和颜色渐变一样进入样式系统。

### 原则 12：对象是关系节点

对象可以依赖其他对象，也可以基于自身状态发生变化。

### 原则 13：动画底座需要前置

动画编辑器可以后置，但参数播放、基础动画和动画数据保存应尽早考虑。

### 原则 14：坐标系保持数学清晰

Plane 不采用手绘 / 艺术坐标系。艺术效果主要由对象和样式产生。

### 原则 15：导出必须形成闭环

PNG、透明背景和高分辨率导出是数学艺术软件的重要闭环。

## 20. 一句话总结

当前 Plane Calculator 的完整设计可以概括为：

> 一个默认界面克制的二维数学艺术画布。  
> 用户可以用函数、几何、列表、参数和 Apple Pencil 创建数学对象，也可以把手绘曲线转化为路径、点列或拟合曲线；多条曲线可以组成可填色区域，系统支持智能封口、颜色与透明度渐变、艺术笔触、对象依赖和基础动画。  
> Plane 负责二维呈现，通用输入、表达式、样式、动画、保存和导出能力应尽量由共享系统承担。
