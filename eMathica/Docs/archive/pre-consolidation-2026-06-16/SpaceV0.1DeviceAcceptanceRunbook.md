# Space v0.1 真机验收手册

## 1. 测试环境

- 设备：iPad（优先横屏），可选 Mac 对照运行。
- 系统模式：Light / Dark 都要覆盖。
- 输入方式：触控为主；如可用，补充触控板/鼠标场景。
- 键盘：记录是否连接外接键盘（本轮不强依赖键盘能力）。

失败记录方式（通用）：
- 每条失败至少提供：截图或录屏、复现步骤、设备与系统版本、是否稳定复现。

## 2. Canvas / Camera 验收

### 2.1 Orbit 左右方向
- 前置条件：打开 Space 文档，画布中至少有 1 个对象或坐标轴可见。
- 操作步骤：选择 orbit，水平左右拖动画布。
- 预期结果：旋转方向符合当前产品定义（“手指向右，场景跟随右移手感”），无反向错觉。
- 失败记录方式：录屏标注“手势方向 vs 实际旋转方向”。

### 2.2 Pitch 极点稳定性
- 前置条件：同上。
- 操作步骤：持续向上拖动，再向下拖动，重复多次。
- 预期结果：视角不锁死、不翻转到异常状态；无明显基向量崩坏。
- 失败记录方式：录屏 + 标注触发动作次数。

### 2.3 Pan 正常
- 前置条件：同上。
- 操作步骤：切到 pan 工具，拖动画布。
- 预期结果：目标平移连续、方向一致，无抖动。
- 失败记录方式：录屏 + 描述方向是否反直觉。

### 2.4 Zoom 正常
- 前置条件：同上。
- 操作步骤：捏合缩放（或等价缩放手势）。
- 预期结果：缩放平滑，距离夹紧策略生效（不过近/不过远）。
- 失败记录方式：录屏 + 指出出现跳变的手势点位。

### 2.5 Camera Undo 单步
- 前置条件：有可见相机变化。
- 操作步骤：执行一次连续 orbit 或 pan 或 zoom 手势，随后 Undo。
- 预期结果：该次连续手势作为单个 Undo step 回退。
- 失败记录方式：录屏 + Undo 次数统计。

### 2.6 亮暗主题可读性
- 前置条件：同一文件切换 Light / Dark。
- 操作步骤：分别观察背景、轴线、对象线条、平面填充。
- 预期结果：Light 背景为浅色；Dark 保持高对比；均可读。
- 失败记录方式：两张对照截图（Light 与 Dark）。

### 2.7 坐标轴与平面可见性
- 前置条件：至少含一个 plane3D。
- 操作步骤：旋转不同角度观察 X/Y/Z 轴与平面。
- 预期结果：轴线明显；plane3D 在 fill/border/grid 维度可感知，不“消失”。
- 失败记录方式：录屏 + 指明问题角度。

## 3. 基础对象创建

### 3.1 point3D 创建
- 前置条件：切到 point3D 工具。
- 操作步骤：单击画布创建点。
- 预期结果：创建成功并自动选中新对象。
- 失败记录方式：截图（对象区 + 画布）。

### 3.2 segment3D 两点创建
- 前置条件：切到 segment3D 工具。
- 操作步骤：两次点击创建线段。
- 预期结果：生成 segment3D 并自动选中。
- 失败记录方式：录屏。

### 3.3 line3D 两点创建
- 前置条件：切到 line3D 工具。
- 操作步骤：两次点击创建直线。
- 预期结果：生成 line3D 并自动选中。
- 失败记录方式：录屏。

### 3.4 plane3D 三点创建
- 前置条件：切到 plane3D 工具。
- 操作步骤：三次点击创建平面。
- 预期结果：三点不共线时创建成功并自动选中。
- 失败记录方式：录屏。

### 3.5 共线点拒绝
- 前置条件：plane3D 工具。
- 操作步骤：输入三点共线组合。
- 预期结果：不创建 plane3D。
- 失败记录方式：录屏 + 点位描述。

### 3.6 过近点拒绝
- 前置条件：plane3D 工具。
- 操作步骤：连续点击极近点位。
- 预期结果：不创建无效 plane3D。
- 失败记录方式：录屏。

### 3.7 创建 Undo/Redo
- 前置条件：任意创建成功后。
- 操作步骤：Undo，再 Redo。
- 预期结果：对象创建可完整回退与重做。
- 失败记录方式：录屏 + 对象数量变化说明。

## 4. Hit Test / Selection

### 4.1 select 点选 point3D
- 前置条件：存在 point3D，切到 select。
- 操作步骤：点击点附近。
- 预期结果：选中对应 point3D。
- 失败记录方式：截图（高亮状态）。

### 4.2 select 点选 segment3D / line3D
- 前置条件：存在 segment3D、line3D，切到 select。
- 操作步骤：点击线附近。
- 预期结果：命中并选中对应对象。
- 失败记录方式：录屏。

### 4.3 空白清空选择
- 前置条件：已有选中对象。
- 操作步骤：点击空白区域。
- 预期结果：选择清空。
- 失败记录方式：截图前后对比。

### 4.4 高亮稳定性
- 前置条件：点、线对象各至少一个。
- 操作步骤：来回切换选择对象。
- 预期结果：selected highlight 正常且不污染对象样式。
- 失败记录方式：录屏。

### 4.5 orbit/pan 不误选
- 前置条件：切到 orbit 或 pan。
- 操作步骤：拖动画布。
- 预期结果：不触发对象选择变化。
- 失败记录方式：录屏 + 标注工具状态。

## 5. Snapping

### 5.1 segment3D snap existing point3D
- 前置条件：已有 point3D，切到 segment3D。
- 操作步骤：在点附近点击作为第一端/第二端。
- 预期结果：端点坐标复用 existing point3D 坐标。
- 失败记录方式：Inspector 坐标截图。

### 5.2 line3D snap existing point3D
- 前置条件：已有 point3D，切到 line3D。
- 操作步骤：两次点击都尽量命中已有点。
- 预期结果：line3D 使用 snapped 坐标构造方向向量。
- 失败记录方式：Inspector 截图 + 操作录屏。

### 5.3 未命中 fallback 工作平面
- 前置条件：切到 segment3D 或 line3D。
- 操作步骤：在远离已有点位置点击。
- 预期结果：回退当前工作平面投影点创建。
- 失败记录方式：记录当前工作平面与结果坐标。

### 5.4 不建立 dependency（当前预期）
- 前置条件：通过 snapping 创建对象后。
- 操作步骤：查看对象 dependency/status。
- 预期结果：不建立父子 dependency，仅复用坐标。
- 失败记录方式：对象详情截图。

### 5.5 无 snap 指示器（当前预期）
- 前置条件：snapping 场景。
- 操作步骤：观察创建过程。
- 预期结果：无可视 snap indicator，属 v0.1 预期。
- 失败记录方式：若出现异常提示，截图说明。

## 6. Work Plane（XY / YZ / ZX）

### 6.1 XY 平面
- 前置条件：切换到 XY。
- 操作步骤：创建 point3D。
- 预期结果：z = 0（允许浮点微小误差）。
- 失败记录方式：Inspector 坐标截图。

### 6.2 YZ 平面
- 前置条件：切换到 YZ。
- 操作步骤：创建 point3D。
- 预期结果：x = 0（允许浮点微小误差）。
- 失败记录方式：Inspector 坐标截图。

### 6.3 ZX 平面
- 前置条件：切换到 ZX。
- 操作步骤：创建 point3D。
- 预期结果：y = 0（允许浮点微小误差）。
- 失败记录方式：Inspector 坐标截图。

### 6.4 segment/line fallback 使用当前工作平面
- 前置条件：存在未命中 snapping 的创建点击。
- 操作步骤：分别在 XY/YZ/ZX 下创建 segment 或 line。
- 预期结果：fallback 点落在当前工作平面。
- 失败记录方式：Inspector + 当前模式截图。

### 6.5 切换工作平面清空 pending
- 前置条件：segment/line/plane 创建进行到中间状态（pending）。
- 操作步骤：切换工作平面后继续点击。
- 预期结果：旧 pending 被清空，不发生跨平面“接续误创建”。
- 失败记录方式：录屏。

### 6.6 runtime-only 预期
- 前置条件：切换到非默认工作平面。
- 操作步骤：保存、关闭、重开文件。
- 预期结果：当前阶段不要求工作平面持久化（runtime-only）。
- 失败记录方式：重开前后截图。

## 9. Delete / Recovery

### 9.1 删除 + Undo（point3D）
- 前置条件：存在 point3D。
- 操作步骤：删除对象后 Undo。
- 预期结果：对象恢复。
- 失败记录方式：录屏。

### 9.2 恢复已删除对象（segment3D）
- 前置条件：删除 segment3D，history 有记录。
- 操作步骤：从“恢复已删除对象”恢复。
- 预期结果：恢复成功并进入 active objects。
- 失败记录方式：截图（history 与对象区）。

### 9.3 plane3D 恢复后独立性
- 前置条件：删除并恢复 plane3D。
- 操作步骤：查看恢复对象属性。
- 预期结果：按当前策略恢复为独立对象。
- 失败记录方式：Inspector 截图。

### 9.4 history 持久化
- 前置条件：有删除记录。
- 操作步骤：保存 -> 关闭 -> 重开。
- 预期结果：deleted history 仍存在（最多 200 条策略）。
- 失败记录方式：history 列表截图。

### 9.5 不破坏 3D geometry
- 前置条件：恢复 3D 对象后。
- 操作步骤：观察画布/Inspector/保存重开。
- 预期结果：3D 几何信息可用，不崩溃。
- 失败记录方式：录屏。

## 10. 性能与交互观察

建议基准（手工观察）：
- 20 个对象：orbit/pan/zoom 是否流畅。
- 50 个对象：首页 preview 更新是否明显卡顿。
- 高频切换选择：highlight 是否稳定。
- Inspector 切换：是否有明显卡顿。
- Light/Dark 切换：画布与对象可读性是否稳定。

失败记录方式：
- 录屏 + 设备型号 + 系统版本 + 对象数量。
